This repository documents a modular homelab and embedded systems environment, with a focus on:

- small form‑factor compute nodes (minilab)
- networking infrastructure
- iterative, experiment‑driven development
- setup & configuration of homelab services
- home automation
- all the rest

The documentation is written as a combination of **build logs**, **reference pages**, **system overviews** and a lot of **Brain dumps** intended to be useful both during active development and as long‑term technical memory.

---

## What You’ll Find Here

### Build Logs

Records of experiments, iterations, and decisions.
These focus on *what changed*, *what broke*, and *what was learned*.

### Hardware & Infrastructure

Canonical reference pages for the hardware used.

### Software & Services

Configuration and setup pages for the various services deployed.

## Documentation Structure

A few guiding principles shape this repository:

- **Build logs** describe *process and change*
- **Hardware pages** describe *components*
- **Software pages** describes *services*

If you’re new, start with:

- Just clicking random pages ;)

## Status & Intent

This is an **active, evolving** system.

Not everything here is finished, optimized, or final—but everything is documented with intent. Decisions, trade‑offs, and failures are captured deliberately, so future changes are informed rather than rediscovered.

The goal is not polish—it’s **clarity, traceability, and understanding**.

I’m figuring this out as I go, expect rough edges, wrong turns, and notes that exist mainly so future‑me
this is meant as a personal reference and a source of inspiration, not a **definitive guide**.
